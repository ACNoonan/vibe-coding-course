# CLI Services

**Keep this page open while you work.** It shows the daily loop and deeper commands when you need them.

---

## Daily Workflow

```bash
git pull
supabase db diff   # if schema changed
npm run dev        # or vercel dev
# build feature with Cursor
git add .
git commit -m "feat: ..."
git push
vercel --prod      # when ready for prod
```

## Git Essentials

### Core Commands

| Command | Purpose |
|---------|---------|
| `git pull` | Sync before you touch code |
| `git status` | Check what changed |
| `git add .` | Stage all changes |
| `git commit -m "..."` | Save grouped work |
| `git push` | Share with GitHub / trigger Vercel |

### Common Fixes

| Situation | Command |
|-----------|---------|
| Merge conflict | `git pull`, resolve files, `git add .`, `git commit` |
| Undo uncommitted file | `git restore path/to/file` |
| Fix last commit message | `git commit --amend -m "new message"` |

<details>
<summary>First-time repo setup</summary>

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/yourusername/your-repo.git
git push -u origin main
```

</details>

## Supabase CLI

### Setup

```bash
supabase login
supabase link --project-ref <ref>
```

### Common Commands

| Command | Purpose |
|---------|---------|
| `supabase migration new <name>` | Scaffold SQL file |
| `supabase db push` | Apply migrations to remote |
| `supabase db pull` | Sync remote schema back to files |
| `supabase db diff` | Compare local vs. remote (requires Docker) |
| `supabase db shell` | Run ad-hoc SQL queries |

### Quick Migration Workflow

```bash
supabase migration new add_tasks_table
# edit the generated SQL
supabase db push
git add supabase/migrations
git commit -m "Add tasks table"
```

<details>
<summary>Local development commands</summary>

```bash
supabase start     # spins up local stack (needs Docker)
supabase db reset  # apply migrations locally
supabase stop      # shut down local services
```

</details>

## Vercel CLI

### Setup

```bash
npm install -g vercel
vercel login
```

### Common Commands

| Command | Purpose |
|---------|---------|
| `vercel` | Link project + deploy preview |
| `vercel --prod` | Push to production manually |
| `vercel dev` | Run Next.js locally with Vercel settings |
| `vercel logs <url>` | View production/runtime errors |

<details>
<summary>Environment variables and settings</summary>

```bash
vercel pull              # Download project settings
vercel env pull .env     # Get env vars locally
vercel ls                # List projects
```

</details>

## Practice Loop

Test your setup with this sequence:

1. `git pull`
2. `supabase migration new add_example`
3. Add a column in the SQL file
4. `supabase db push`
5. `git add supabase/migrations`
6. `git commit -m "Add example column"`
7. `git push`
8. `vercel --prod`

If any step fails, capture the error and check [Troubleshooting](/troubleshooting).
